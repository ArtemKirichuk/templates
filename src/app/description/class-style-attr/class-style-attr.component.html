Привязка через атрибуты позволяет улучшить удобство, придать динамику через стили, и управлять множеством css классов. Рекуомендуется использовать привязку через свойства где это возможно в остальных случаях используется привязка через птрибуты.Для примера aria и svg это чистые атрибуты. Ни aria ни svg не содержит свойств.

Синтаксис

Привязка атрибутов напоминает привязку свойств с предшествующим словом "attr.". Если выражение с права соответствует undefined или null то Angular удаляет такие атрибуты.

Директиву NgStyle можно использовать как альтернативу прямым привязкам [style]. Однако предпочтительнее использовать [style]. В Angular NgStyle больше не имеет существенного значения и может быть удален в будущем.

Приоритет стилей

В шаблоне

  • [class.foo]="hasFoo"
  • [class]="classExpr"
  • class="foo"

Директива host

  • {{"host: {'[class.foo]': 'hasFoo'}"}}
  • {{"host: {'[class]': 'classExpr'}"}}
  • {{"host: host: {'class': 'foo'}"}}

Компонент host

  • {{"host: {'[class.foo]': 'hasFoo'}"}}
  • {{"host: {'[class]': 'classExpr'}"}}
  • {{"host: {'class': 'foo'}"}}

Чем конкретнее привязка класса или стиля, тем выше его приоритет.

Ввод в конструктор значения атрибута
