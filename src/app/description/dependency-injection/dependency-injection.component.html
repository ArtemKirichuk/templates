Сервисы - объекты класса необходимые для выполнения функций.Dependency injection(DI) - это шаблон разработки в котором класс запросов зависимостей из внешнего источника.

Создание injectable сервиса

  • Создаём сервис через cli ng g s name . Данная команда создаст injectable класс, который будет доступен по всему приложению
  • Далее создаём метод для получения тестовы данных
  • Далее вводим данный сервис в конструктор компонента где он будет использоваться

рекомендуется определять сервисы и компоненты в разных файлах. При определениии сервиса и компонента в одном фале сервис должен определяться первым.
